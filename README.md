# Live Notify PWA

PWA-приложение для iPhone и Android: пользователь открывает ссылку, включает уведомления и получает push, когда вы выходите в эфир.

## Что умеет
- подписка на web push;
- адаптация интерфейса под iPhone и Android;
- инструкции по установке прямо на странице;
- кнопка **«Я в эфире»** для массовой рассылки;
- хранение подписчиков в SQLite;
- PWA-установка на экран устройства.

## Важно для iPhone
На iPhone push работают для **Home Screen web apps**: пользователь должен открыть сайт в Safari, выбрать **«На экран Домой»**, затем открыть приложение с экрана Домой и уже там разрешить уведомления. ## Запуск

### 1. Установите зависимости
```bash
npm install
```

### 2. Сгенерируйте VAPID-ключи
```bash
npx web-push generate-vapid-keys
```

### 3. Создайте `.env`
```bash
cp .env.example .env
```

Заполните:
- `ADMIN_SECRET` — ваш секрет для отправки уведомлений;
- `LIVE_URL` — ссылка на эфир по умолчанию;
- `VAPID_PUBLIC_KEY` и `VAPID_PRIVATE_KEY`;
- `VAPID_SUBJECT` — например `mailto:you@example.com`.

### 4. Запустите проект
```bash
npm start
```

### 5. Откройте по HTTPS
Для реальных push нужен HTTPS. Локально можно тестировать часть функций, но рабочая подписка и уведомления обычно требуют HTTPS. Для продакшена подойдёт Render, Railway, VPS с Nginx или любой хостинг с SSL. ## Как пользоваться
### Участник на iPhone
1. Открывает ссылку в Safari.
2. Нажимает **Поделиться** → **На экран Домой**.
3. Открывает иконку приложения с экрана Домой.
4. Нажимает **Включить уведомления**.

### Участник на Android
1. Открывает ссылку в Chrome.
2. Нажимает **Включить уведомления**.
3. По желанию устанавливает приложение на экран.

### Автор
1. Открывает ту же страницу.
2. Вводит `ADMIN_SECRET`.
3. Проверяет ссылку на эфир.
4. Нажимает **«Я в эфире»**.
